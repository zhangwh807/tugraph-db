# Bolt client

TuGraph implements Neo4j's bolt protocol, you can use Neo4j's client to access TuGraph.

## Enable bolt port

If you are using the TuGraph runtime docker image, the config file is `/usr/local/etc/lgraph.json`. If Tugraph is deployed using an RPM package, the configuration file is located on the server at `/usr/local/etc/lgraph.json`.

In the Tugraph configuration file, add bolt_port=7687, which is the default value. If you need to change it, please modify it in the configuration file accordingly. After making any changes, please restart Tugraph to open the bolt port. The Neo4j client will need to connect to this port. For specific instructions on restarting the service, please refer to the [Tugraph Running](../../5.installation&running/7.tugraph-running.md) . For detailed configuration options in the configuration file, see the Service Configuration section in [Tugraph Running](../../5.installation&running/7.tugraph-running.md) .

## Use case example

Add Maven dependency

```xml
<dependency>
    <groupId>org.neo4j.driver</groupId>
    <artifactId>neo4j-java-driver</artifactId>
    <version>4.4.2</version>
</dependency>
```

The client is instantiated in the following format.

```java
    Driver driver = GraphDatabase.driver("bolt://ip:port", AuthTokens.basic("admin", "73@TuGraph"));
```

Common phrases

```java
        Session session = driver.session(SessionConfig.forDatabase("default"));//Create a session through the driver object, setting the session to connect to a specific database, to execute Cypher statements.
        session.run("CALL db.dropDB()");//Clear the graph project, please do not attempt this lightly as it will delete both the schema and data of the selected graph project.
        session.run("CALL db.createVertexLabel('person', 'id' , 'id' ,INT32, false, 'name' ,STRING, false)");//Create vertex
        session.run("CALL db.createEdgeLabel('is_friend','[[\"person\",\"person\"]]')");//Create edge
        session.run("CALL db.addIndex(\"person\", \"name\", false)");//Create index
        session.run("create (n1:person {name:'jack',id:1}), (n2:person {name:'lucy',id:2})");//Insert vertex data
        session.run("match (n1:person {id:1}), (n2:person {id:2}) create (n1)-[r:is_friend]->(n2)");//Insert edge data
        Result res = session.run("match (n)-[r]->(m) return n,r,m");//Query vertices and edges
        session.run("match (n1:person {id:1}) delete n1");//Delete vertex data
        session.run("match (n1:person {id:1})-[r]-(n2:person{id:2}) delete r");//Delete edge data
        session.run("CALL db.deleteLabel('edge', 'is_friend')");//Delete edge
        session.run("CALL db.deleteLabel('vertex', 'person')");//Delete vertex
```

Details on the use of Cypher and stored procedures can be found in [Cypher](../../8.query/1.cypher.md)

## Limitations on use

Currently, not all features of the Neo4j Bolt protocol are supported. The transaction display feature of the Neo4j client and the property of a weak schema are two functionalities that are not yet supported. Please be cautious and avoid using these two features when utilizing the client.
